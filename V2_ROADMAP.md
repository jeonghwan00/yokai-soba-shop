# V2 Roadmap — 요괴 소바집

V1 (Cursor Vibe Jam 2026 출품작)은 7일짜리 단편 캠페인이다. V2의 목표는
*시간 들여 키울 이유*를 만드는 것 — 미스티아의 이자카야 / 스타듀처럼
"그만두기 어려운" 코지 경영 시뮬로 진화시키기.

V1이 *씨앗*이라면 V2는 *정원*. 아래 다섯 기둥이 동시에 굴러가야 진짜
"몇 시간씩 플레이" 가능한 게임이 된다.

---

## 1. 호감도가 해금을 풀어야 한다 (가장 중요)

V1: 호감도는 숫자에 불과. 31/71에서 대사 톤만 바뀜.

**V2 단계별 해금**:
- **30**: 그 캐릭터의 *정규 단골 자리* — 매 밤 자동 등장 (캠페인 큐 고정에서 풀림)
- **60**: *사이드 스토리* — 다음 아침 회상에 그 캐릭터의 과거 한 토막
- **90**: *기념품* — 그 캐릭터가 가게에 영구 오브젝트 남김 (벽에 깃털, 카운터에 손수건…). 야간 배경에 시각적으로 누적

**기술 노트**:
- `state.affinity[key]`는 이미 있음. 추가로 `state.unlocks[key] = ['regular','sideStory','keepsake']` 같은 배열 필요
- 기념품은 새 스프라이트 + 야간 씬 렌더에서 좌표별로 그림
- 사이드 스토리는 `data/sideStories.js`에 캐릭터별 스크립트 — 트리거 시 morningScene이 평소 회상 대신 사이드 스토리 재생

---

## 2. 돈이 가게를 키워야 한다

V1: 누적 매출은 통계에 불과. 캠페인 끝나면 사라짐.

**V2 상점 시스템** — 영업 종료 후 정산 화면에서 구매:
- **노렌 색깔**: 빨강/감색/먹색 등 — 가게 외관 변경
- **랜턴**: 일반/제등/등명 — 호감도 획득률 +5% 수준의 패시브
- **라디오 트랙**: BGM 교체 (ambient/rain/koto 변형 등)
- **재료 슬롯**: 잠금 해제 시 새 식재료 활성화 → 새 레시피 가능
- **카운터 의자**: 내구성/장식 — 더 많은 동시 손님 지원 (V2.1)

**기술 노트**:
- `state.shop = { noren: 'red', lantern: 'standard', track: 'main', unlockedSlots: ['default'] }` 추가
- 정산 화면에 상점 탭 추가
- 야간 씬이 `state.shop`을 읽어 배경/오브젝트 합성

---

## 3. 레시피는 발견되어야 한다

V1: 12개 레시피가 첫 날부터 노트북에 다 있음.

**V2 발견 흐름**:
- 시작 시 **4개**만 알고 있음 — 할머니가 자주 만들던 기본 (차가운 메밀국수, 카케 우동, 카파마키, 두부탕)
- 캐릭터 호감도 60 → 그 캐릭터의 *비밀 레시피* 노트북에 추가
- 새 재료 슬롯 해금 시 그 재료가 들어간 레시피 잠금 해제
- *우연한 조합* 실험으로 발견 — 잘못 만든 게 새 레시피로 등록 (예: 김 + 두부 + 파 → 김두부 발견)

**기술 노트**:
- `state.discoveredRecipes` 배열 — 노트북은 이 안에 있는 것만 표시
- `evaluateBowl`은 모든 레시피 매칭은 그대로 — 새로 매칭된 것만 발견됨 처리
- 캐릭터 favoriteRecipe 도달 안 했어도 다른 매칭 레시피 발견 가능

---

## 4. 엔드리스 모드 — 7일이 끝이 아님

V1: 7일차 → 엔딩 → 처음으로.

**V2 "매일 밤" 모드** — 엔딩 후 해금:
- 손님 랜덤 등장 (기존 5명 + 새 손님들)
- **새 손님들**: 잇탄모멘 (걸어다니는 천), 죠로구모 (거미 여인), 카라카사 (우산 요괴), 라이주 (벼락의 짐승), 텐조나메 (천장을 핥는 자) 등
- 난이도 점진 상승: 단골일수록 모호한 주문 비중↑
- 매일 *오늘의 특별 메뉴* — 보너스 매출
- 진행 목표:
  - 모든 호감도 90+
  - 모든 기념품 수집
  - 모든 레시피 발견
  - 가게 풀 업그레이드

**기술 노트**:
- 새 모드는 `state.scene = SCENES.ENDLESS_NIGHT` (or reuse NIGHT with `state.endless = true`)
- 캠페인 데이터 분리 — `data/campaign.js` (V1) + `data/endlessPool.js` (V2 손님 풀)
- localStorage에 *완전히* 저장 (V1은 호감도/일차만 저장)

---

## 5. 캐릭터 사이드 스토리 (감동의 원천)

V1: 7일 일자형 흐름 + 일반 대사. 캐릭터의 과거가 거의 안 풀림.

**V2 호감도 60에서 풀리는 회상**:

각 캐릭터에 *그분의 과거* 한 컷씩. 노트북의 새 탭 "이야기 페이지"에 영구 저장됨.

- **유키온나**: "그분이 흘린 눈물 — 사실 사람이었을 때, 잃어버린 아이를 위해서였단다."
- **키츠네**: "그 외상 — 사실 동생 분 약값이었지. 결국 나았더라."
- **캇파**: "강이 메말라가던 시절, 할머니의 다시가 그분을 살렸단다."
- **텐구**: "수백 년 전, 할머니가 첫 영업하던 날 — 그분이 첫 손님이셨다."
- **네코마타**: "버려진 새끼고양이를 — 할머니가 거두셨지. 그게 50년 전이란다."
- **할머니**: 사이드 스토리 대신 *최종 비밀* — 츠키미 우동의 진짜 의미 (V1엔 그냥 "마지막 약속"이지만 V2엔 더 깊은 사연 풀림)

**기술 노트**:
- `data/sideStories.js`: `{ characterKey: { lines: [...], unlockedAt: 60 } }`
- 호감도 도달 즉시 알림 ("이야기가 한 페이지 추가됐다")
- 다음 아침 회상이 사이드 스토리로 대체 (1회성)
- 노트북에 "이야기" 탭 추가 — 모인 페이지 모음

## 구현 우선순위 (V2.0 → V2.1 → V2.2)

| 패치 | 항목 | 비용 |
|---|---|---|
| **V2.0** | 호감도 해금 + 사이드 스토리 | 8h |
| **V2.0** | 레시피 발견 시스템 | 4h |
| **V2.0** | 상점 — 노렌/랜턴/라디오 | 6h |
| **V2.1** | 엔드리스 모드 + 새 손님 3명 | 10h |
| **V2.1** | 다중 엔딩 (호감도 조합 분기) | 4h |
| **V2.2** | 포레징 / 식재료 채집 미니게임 | 10h |
| **V2.2** | 가게 꾸미기 (벽지/의자/소품) | 8h |

**V2.0 첫 패치 (~18시간) 추천**: 호감도 해금 + 사이드 스토리 + 상점. 이 셋만 들어가도 "아 이거 좀 키워볼만 하네" 감각.

## 추가 아이디어 (시간 남으면)

- **계절 시스템**: 봄/여름/가을/겨울 — 각 계절에만 오는 손님
- **선물 시스템**: 캐릭터별 좋아하는 아이템 — 호감도 +
- **요리 도전**: NPC가 보내는 우편 — 며칠 내 특정 요리 N그릇 서빙
- **음악 시퀀서**: 라디오에서 직접 BGM 선곡
- **포토 모드**: 가게 사진 찍어 저장
- **할머니의 일기**: 매일 한 페이지씩 풀리는 일기책 (별도 콜렉션)

---

## V1 → V2 마이그레이션 노트

V1 저장 데이터 (`localStorage.yss_*`):
- `yss_locale`: 언어 설정 (유지)

V2에서 추가:
- `yss_state`: 전체 게임 상태 직렬화 (호감도, 해금, 상점, 발견 레시피, 일차 등)
- `yss_keepsakes`: 보유 기념품 목록
- `yss_storyPages`: 풀린 사이드 스토리 ID 목록

V1 → V2 첫 실행 시 마이그레이션: 기존 `yss_locale`만 유지, 나머지는 새로 시작.
